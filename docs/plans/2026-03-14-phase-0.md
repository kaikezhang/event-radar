## Task 1: Unify Event Taxonomy

### Problem
- `llm-classification.ts`: `eventType: z.string()` (free-form)
- `llm-types.ts`: `LLMEventTypeSchema` enum has only 7 values: `filing`, `earnings`, `insider`, `macro`, `political`, `analyst`, `social`
- Scanners emit wild values: `8-K`, `form-4`, `halt`, `resume`, `social-volume`, `economic-release`
- `event-type-mapper.ts` skips `fda`, `congress`, `doj-antitrust`, `whitehouse` entirely

### Solution
Expand the enum to ~20 types that cover all scanners:

```typescript
// packages/shared/src/schemas/llm-types.ts
export const EventTypeSchema = z.enum([
  // Earnings
  'earnings_beat',
  'earnings_miss',
  'earnings_guidance',
  
  // SEC / Corporate Filings
  'sec_form_8k',
  'sec_form_4',  // insider trading
  'sec_form_10q',
  'sec_form_10k',
  
  // Regulatory
  'fda_approval',
  'fda_rejection',
  'fda_orphan_drug',
  'ftc_antitrust',
  'doj_settlement',
  
  // Government
  'executive_order',
  'congress_bill',
  'federal_register',
  
  // Macro
  'economic_data',
  'fed_announcement',
  
  // Technical / Flow
  'unusual_options',
  'insider_large_trade',
  'short_interest',
  
  // Sentiment
  'social_volume_spike',
  'reddit_trending',
  'news_breaking',
]);
```

### Files to Modify
1. `packages/shared/src/schemas/llm-types.ts` — Expand enum
2. `packages/shared/src/schemas/llm-classification.ts` — Use unified enum
3. All scanners — Map source values to unified types
4. `packages/backend/src/pipeline/event-type-mapper.ts` — Remove exclusions
5. `packages/backend/src/pipeline/llm-gatekeeper.ts` — Update prompt to classify into new types
6. `packages/delivery/*` — Update display labels

### Sub-task: Add Subtypes (Optional v2)
Each top-level type can have subtypes:
```typescript
const EventSubtypeSchema = z.object({
  type: EventTypeSchema,
  subtype: z.string().optional(), // e.g., "fda_approval" + "accelerated"
});
```

```typescript
// packages/backend/src/services/outcome-tracker.ts
const OUTCOME_INTERVALS = [
```

## Task 3: Fix LLM Output: Chinese → English

### Problem
- `llm-enricher.ts` prompt asks for Chinese output:
  - `"summary": "1-2 sentence Chinese summary"`
  - `"impact": "1-2 sentences ... (Chinese)"`
  - `"regimeContext": "... (Chinese)"`
- Action labels are Chinese emoji: `🔴 立即关注`, `🟡 持续观察`, `🟢 仅供参考`

### Solution

**Step 1: Update action labels to English**
```typescript
// packages/shared/src/schemas/llm-types.ts
export const LLMEnrichmentActionSchema = z.enum([
  '🔴 ACT NOW',      // Was: 立即关注
  '🟡 WATCH',        // Was: 持续观察  
  '🟢 FYI',          // Was: 仅供参考
]);
```

**Step 2: Update enrichment prompt**
```typescript
// packages/backend/src/pipeline/llm-enricher.ts
const ENRICHMENT_PROMPT = `

Respond in JSON with exactly these fields:
{
  "summary": "1-2 sentence English summary of what happened",
```

## Task 4: Document Scanner Latency

### Problem
No one knows which scanners are fast vs slow. Product makes claims about "seconds after event" but:
- Congress scanner: 30 min
- White House: 15 min
- FDA: 5 min
- Reddit: 1 min
- Truth Social: 15 sec

### Solution

Create `docs/SCANNERS.md`:

```markdown
# Scanner Latency Reference

| Scanner | Poll Interval | Typical Latency | Notes |
|---------|---------------|-----------------|-------|
| Truth Social | 15s | ~20s | Fastest real-time |
| X (Twitter) | 30s | ~45s | |
| Reddit | 60s | ~1-2min | |
| StockTwits | 60s | ~1-2min | |
| Newswire | 2min | ~2-5min | |
| FDA | 5min | ~5-10min | |
| White House | 15min | ~15-20min | |
| Congress | 30min | ~30-45min | Slowest |
| SEC EDGAR | 30s | ~30-60s | RSS-based |

## SLA Targets

**For swing traders:**
- All scanners < 15 min — acceptable
- Critical sources (FDA, SEC, White House) < 5 min — target

**Day traders (future):**
- Requires WebSocket feeds or pusher integration
- Not supported in MVP
```

## Task 5: Fix Source Naming Inconsistency

### Problem
- Scanner emits `source: 'x'`
- Downstream expects `x-scanner`
- Causes display and filter issues

### Solution

1. Audit all scanners for `source` field
2. Create mapping table in `scanner-registry.ts`
3. Normalize at scanner registration time

```typescript
// packages/shared/src/scanner-registry.ts
const SOURCE_ALIASES: Record<string, string> = {
  'x': 'x-scanner',
  'twitter': 'x-scanner',
  'form-4': 'sec-edgar-scanner',
  '8k': 'sec-edgar-scanner',
};
```
