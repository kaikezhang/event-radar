## Summary

---

## Issues Found

### P1 — Critical

#### C1: Bottom nav bar overlaps page content on EVERY page
- **Where:** All pages — Feed, Watchlist, Scorecard, History, Settings, Onboarding
- **What:** The sticky bottom navigation bar (Feed/Watchlist/Scorecard/History/Settings) sits on top of page content without adequate bottom padding. Content scrolls behind the nav but the last ~60px of scrollable content is permanently obscured.
- **Impact:** Users cannot see or interact with content hidden behind the nav. On onboarding, the "Enable notifications" CTA button is partially hidden. On Settings, the "PERMISSION: DENIED" banner is cut off. On Watchlist, the $MUR ticker row is obscured.
- **Repro:** Visit any page. Scroll to bottom. Content is behind the nav bar.
- **Fix:** Add `padding-bottom: 80px` (or equivalent) to main content containers on all pages.

### P2 — High

#### H1: Event detail tabs (Evidence/Trust) show duplicated or identical content
- **Where:** `/event/:id` — Evidence and Trust tabs
- **What:** Clicking "Evidence" tab shows the same content as "Summary." Clicking "Trust" tab causes the header/tab bar to render twice (duplicate Back button, duplicate Summary/Evidence/Trust tabs), and the content appears to be the same page rendered twice stacked vertically.
- **Impact:** Two of three event detail tabs are broken. Users cannot access differentiated evidence or trust content.
- **Repro:** Open any event → click "Evidence" → same as Summary. Click "Trust" → duplicated UI.

#### H2: Scorecard page stuck on skeleton loading at mobile viewport (375px)
- **Where:** `/scorecard` at 375px width
- **What:** The scorecard page shows only skeleton placeholder boxes and never loads actual data on mobile viewport. Desktop and tablet load fine.
- **Impact:** Mobile users cannot access scorecard at all.
- **Repro:** Set viewport to 375x812. Navigate to `/scorecard`. Observe eternal skeleton.

#### H3: Light mode is partially implemented
- **Where:** All pages when theme is set to "Light"
- **What:** Event cards retain dark backgrounds and light text when the page background switches to white. The result is dark cards floating on a white background — visually jarring and inconsistent.
- **Impact:** Light mode is unusable. Users who switch to light will immediately switch back.
- **Repro:** Settings → Appearance → Light. Navigate to Feed. Observe dark cards on white background.

### P3 — Medium

#### M1: Feed initial load shows skeleton for 3+ seconds
- **Where:** Feed page on both "My Watchlist" and "All Events" tabs
- **What:** Skeleton loading state persists for 3-5 seconds before events render. No loading spinner or progress indicator — just static gray bars.
- **Impact:** Feels slow. Users may think the app is broken.

#### M2: WebSocket connection warning on page load
- **Where:** Console
- **What:** `WebSocket connection to 'ws://localhost:5173/ws/events' failed: WebSocket is closed before the connection is established.` appears on initial load and on navigation.
- **Impact:** May cause missed live updates. The "Live" indicator shows red dot but WebSocket fails silently.

#### M3: History "BY SEVERITY" card shows only "50" with no breakdown
- **Where:** `/history` — stats cards
- **What:** The "BY SEVERITY" card shows a single orange dot and "50" with no label, no severity breakdown (critical/high/medium/low counts), and no context.
- **Impact:** The card is meaningless — it communicates no useful information.

#### M4: Onboarding "Trending this week" section partially hidden
- **Where:** `/onboarding` step 2
- **What:** The "Trending this week" section header and trending tickers are partially obscured by the sticky "You're watching X tickers / Continue" footer bar.
- **Impact:** Users may miss trending tickers they'd want to add.

### P4 — Low

#### L1: Blue dot artifact on onboarding completion screen
- **Where:** `/onboarding` final step ("You're all set!")
- **What:** Small blue dot visible near top of page. Appears to be a stray element or rendering artifact.
- **Impact:** Minor visual polish issue.

#### L2: Mobile feed shows "All Events" while desktop shows "My Watchlist" as default
- **Where:** Feed page at 375px vs 1280px
- **What:** The default feed view differs between mobile and desktop viewports. Mobile defaults to "All Events" while desktop defaults to "My Watchlist."
- **Impact:** Inconsistent experience across devices. Users may not see their watchlist events on mobile.

## What's Working Well

- **Onboarding flow:** Smooth 4-step wizard (Welcome → Tickers → Notifications → Done). Ticker selection with popular tickers, sector packs, and trending tickers is excellent. The 3-ticker minimum with live counter works well.
- **Feed cards:** Clear severity indicators (CRITICAL/HIGH with color-coded dots), BEARISH/BULLISH badges with confidence levels, source attribution, ticker tags, and star/bookmark functionality.
- **Event detail page (Summary tab):** Comprehensive: What Happened, Why It Matters Now, Bull vs Bear Case, Risk Factors, Source Details, Market Regime, Alert Provenance pipeline, Trust/Scorecard verification, feedback buttons.
- **Watchlist management:** Drag-reorder handles, notes field, search, X to remove, "+ New Section" for organization.
- **Scorecard (desktop):** Detailed calibration page with total alerts, hit/worked rates, source accuracy visualization, rolling accuracy trend chart, signal/confidence/source/event type bucket breakdowns.
- **History page:** Date range picker, severity/source/ticker filters, summary stats.
- **Settings:** Theme toggle, push alerts setup with clear 3-step guide, notification budget, sound alerts, audio squawk.
- **Keyboard shortcuts:** Power-user friendly — /, ?, ESC, j/k, arrow keys.
- **Responsive (tablet/desktop):** Clean two-panel layout on desktop, single-column on tablet.
- **Network:** All API requests return 200. No failed requests. Fast response times (10-75ms).
