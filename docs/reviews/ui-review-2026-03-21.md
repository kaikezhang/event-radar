# UI / Visual Design Review — 2026-03-21

**Reviewer perspective:** UI/Visual designer evaluating visual hierarchy, typography, color system, spacing, component consistency, and overall polish.

---

## Visual Hierarchy — 7/10

**What works:**
- Event severity labels (CRITICAL in red, HIGH in orange) immediately draw the eye. The colored dot + text label + type tag is a clear visual system.
- The BEARISH/BULLISH badges with directional arrows (▼/▲) are instantly scannable. The green-for-bullish, red-for-bearish encoding follows financial conventions.
- Feed cards have clear hierarchy: severity → headline → badge → snippet → source → tickers. Top-to-bottom priority is correct.
- The left orange accent bar on selected/highlighted cards is effective.

**What doesn't work:**
- **The event detail page has no visual hierarchy.** Every section (What Happened, Why It Matters, Bull vs Bear, Risk Factors, Source Details, Market Regime, Source Journey, Trust, Feedback) gets equal visual weight — same card style, same heading size, same spacing. Nothing says "THIS is the most important thing." The Bull/Bear thesis should dominate, not sit at equal weight with "Source Details: CNBC."
- **The Scorecard page header** shows four stats (Total alerts: 23866, Directional hit rate: 24.4%, Setup worked rate: 24.4%, Avg T+20 move: 0.0%) in equal-sized boxes. But these are not equally important. The hit rate IS the headline. It should be 3x the size of the others.
- **The History summary cards** (Total: 23,897 / By Severity: 50 / Top Tickers) are visually equal but "Total" is the only one with useful content. "By Severity: 50" is meaningless without context.

---

## Typography — 7/10

**Font:** Inter — excellent choice. Clean, highly legible, designed for UI. Works well at all sizes tested.

**Sizing:**
- Page titles (e.g., "History", "Scorecard"): Good size, clear hierarchy
- Section headers (e.g., "CATALYST / EVENT SUMMARY", "MARKET CONTEXT"): Small caps with labels work well for section categorization
- Body text: Readable at default and mobile sizes
- Card headlines: Bold, appropriately sized for scanning
- Ticker tags ($NVDA, $AAPL): Monospace-style in pills — clear and scannable

**Issues:**
- **Line height on event detail snippets** is too tight. The text in the Bull/Bear case section and Market Context feels cramped. More generous line-height (1.6 instead of ~1.4) would improve readability for the longer text blocks.
- **The "via CNBC" source attribution** is too small and too low contrast (gray on dark). Sources are critical for trust — they should be more visible.
- **Scorecard bucket tables** use a very small font for headers (ALERTS, HIT RATE, WORKED RATE, AVG T+20). At the current size, they're hard to read, especially on the long scorecard scroll.

## Color System — 6/10

### Dark Mode (Primary)

**Background palette:**
- Page background: ~#0a0a0a (very dark, almost pure black)
- Card backgrounds: ~#141414 (slightly lighter)
- Input/section backgrounds: ~#1a1a1a
- This is a good dark mode foundation. Sufficient contrast between layers.

**Accent colors:**
- Orange (#f97316-ish): Used for primary CTAs, selected states, active nav, onboarding progress, severity HIGH labels, links, borders. **Orange is doing too much.** It's the color for EVERYTHING. This makes it hard to distinguish a CTA button from an active filter from a severity indicator from a nav highlight.
- Red: CRITICAL severity, BEARISH badge, source accuracy bar
- Green: BULLISH badge, onboarding completion icon
- Yellow/Gold: Star/bookmark, Medium severity, warning badges

**Problems:**
- **Orange overload.** Count the orange elements on the feed page: nav underline, filter chips, sort dropdown border, progress bar, severity dot (HIGH), card border accent on hover, CTA buttons. When everything is orange, nothing stands out. Consider a secondary color (blue? teal?) for interactive controls, reserving orange strictly for severity/priority.
- **The source accuracy bar on Scorecard is entirely red** with no green. This is probably accurate data, but visually it screams "EVERYTHING IS WRONG." If accuracy is low, show it honestly, but consider adding context (e.g., "based on 20 usable verdicts out of 23,866 events" — the red bar represents a tiny sample).
- **CRITICAL uses red dot, HIGH uses orange dot, MEDIUM uses yellow dot, LOW uses gray dot.** This is a good severity spectrum, but the dots are tiny (6-8px). At a glance, orange and yellow are hard to distinguish.

### Light Mode

**Broken.** The page background switches to white, but event cards retain dark backgrounds. This creates a jarring high-contrast collision. Text within cards appears unchanged. The filter chips, nav bar, and header adapt to light mode, but the primary content (cards) doesn't. Light mode needs either complete implementation or removal.

## Spacing & Layout — 7/10

**Consistent patterns:**
- Card padding: ~16-20px. Consistent across feed cards, watchlist items, scorecard sections.
- Section spacing on event detail: ~16px between sections. Consistent.
- Nav bar height: ~56px. Standard.
- Grid layout on Scorecard summary: 2-column grid for bucket stats. Clean.

**Issues:**
- **Bottom nav overlap:** The #1 spacing issue. The sticky bottom nav (~60px) sits on top of page content with no corresponding bottom padding on the content container. This affects EVERY page.
- **Event detail sidebar** (Market Regime + Quick Actions) floats to the right on desktop but has awkward spacing — it feels disconnected from the main content column. The gap between the main content and sidebar is too large.
- **Watchlist ticker rows** have generous vertical spacing but the "Add note..." placeholder and the X delete button are at opposite ends with nothing in between — lots of dead space.
- **Onboarding ticker grid** has uneven spacing between popular tickers, sector packs, and trending tickers sections.

## Component Consistency — 7/10

**Consistent:**
- Card components (feed cards, watchlist items, history items, scorecard sections) share a common visual language: dark background, subtle border, rounded corners.
- Button styles: Orange filled for primary CTAs, outlined for secondary actions.
- Badge/pill components: Severity badges, ticker tags, filter chips — all use a consistent pill shape.
- Section headers: All use the small-caps label pattern ("CATALYST / EVENT SUMMARY", "RISK FACTORS", "SOURCE DETAILS").

**Inconsistent:**
- **The "Continue" button** in onboarding uses a filled orange pill shape. The "Get started" button on the welcome page uses the same style. But "Skip setup" is plain text with no button styling. The "Maybe later" on the notifications page is also plain text. These secondary actions should have consistent styling (e.g., outlined button).
- **The feed dropdown** ("My Watchlist ▾") is styled as an outlined button, but the sort dropdown ("Latest first ▾") is styled as a form select. These are functionally similar (both are dropdown selectors) but visually different.
- **"View original article →"** in event detail uses an arrow, while "View original source" uses an external link icon (↗). Same action, different visual treatment.

## Data Visualization — 6/10

**The rolling accuracy trend chart (Scorecard):**
- Line chart showing 7-day rolling directional hit rate by severity. The lines are thin and the labels are small. The chart area is tall but the data range (0-100%) means most of the chart is empty above the ~25% accuracy line. Consider zooming the Y-axis to 0-50% for better resolution, or showing a target/benchmark line.
- Legend (Critical, High, Medium, Low) is at the bottom and very small.

**Source accuracy bar (Scorecard):**
- A simple stacked bar showing accuracy %. Currently 100% red. The bar has no labels, no percentage text, no axis markers. Without reading the heading "Source accuracy," you wouldn't know what this bar represents.

**Severity dots (feed/history):**
- Tiny colored dots. Functional but could be more distinctive. Consider using filled/outline icons instead of dots for better differentiation.

**Missing visualizations:**
- No price sparklines on feed cards
- No "events per day" histogram on History
- No visual timeline for Alert Provenance (it's a text list, could be a visual pipeline)

## Icon Usage — 7/10

Icons used: Lucide React icon library. Consistent style (thin line icons).

- Home, Eye, BarChart, Clock, Settings — nav icons are clear and standard
- Bell for notifications, Star for bookmark, Link for external, Share for sharing
- Question mark (?) for keyboard shortcuts — unconventional. Most apps use ⌨️ or don't show it in the header.

**Issues:**
- The "?" button in the top-right header is ambiguous — is it help? FAQ? It opens keyboard shortcuts, which is unexpected.
- The settings gear (⚙️) in the top-right is redundant with "Settings" in the bottom nav. Two ways to reach the same page.

## Top 5 Visual Improvements (Ranked by Polish Impact)

### 1. Fix bottom nav padding across all pages
Every page has content hidden behind the nav. This is the most visible polish issue. One CSS rule change fixes the most pages simultaneously.

### 2. Complete light mode or remove the option
A broken light mode is worse than no light mode. Either invest in proper theming (CSS variables for all card/component backgrounds) or hide the toggle until it's ready.

### 3. Reduce orange overload — introduce a secondary accent color
Orange is used for severity, CTAs, active states, links, borders, and progress indicators. Introduce blue or teal for interactive controls (buttons, links, active states), and reserve orange for severity/priority signaling.

### 4. Add visual hierarchy to the event detail page
Make the headline and directional thesis visually dominant. Reduce the visual weight of secondary sections (source details, market regime). Use card elevation or background variation to create depth.

### 5. Add labels and context to Scorecard visualizations
The source accuracy bar needs percentage labels. The rolling accuracy chart needs a larger legend and target line. Bucket tables need consistent formatting when data is "N/A" — either gray them out or collapse them.
