## User Flow Analysis

### Primary Journey: "I want to see what's happening with my stocks"

```
Open app → Feed (Smart Feed/Watchlist) → Scan cards → Click event → Read detail → Back to feed
```

**v1 friction points — resolved:**
1. ~~Clicking an event on mobile loses scroll position~~ — not retested but detail panel now works in split-view on desktop, maintaining feed context.
2. ~~No way to quickly scan AI analysis~~ — BEARISH/BULLISH badges with "High conf" are now visible on cards. Still no one-line thesis preview on all cards though.
3. ~~Event detail tabs broken, everything on Summary~~ — **FIXED.** Summary, Evidence, and Trust now show distinct content. This is the biggest UX win of all 6 sprints.

**New friction point:**
- Smart Feed vs My Watchlist vs All Events — three modes is good for power users but adds a decision point for new users. The default depends on how you arrived (onboarding lands on "My Watchlist" via `?tab=watchlist`). The "What is Smart Feed?" tooltip helps, but a first-time user still has to understand three modes before they can engage.

### Secondary Journey: "Set up my alerts"

```
Onboarding → Pick tickers → Skip notifications → Feed
(Later) Settings → See "permission denied" → Read recovery steps → Fix browser settings
```

**v1 friction points — status:**
1. ~~Settings shows "PERMISSION: DENIED" with no guidance~~ — **IMPROVED.** Now shows a 3-step recovery guide with platform-specific instructions. "Click the lock icon → Find Notifications → Change to Allow → Refresh." This is genuinely helpful. However, the raw "permission denied" text still appears above the guidance, creating a mixed signal.
2. ~~No fallback channels~~ — Still no email/Telegram/Discord alternative visible in Settings. The backend has Discord webhooks but they're not exposed in the UI.

### Tertiary Journey: "Can I trust these signals?"

```
Feed → Click event → Evidence tab → Trust tab → Source Journey → Scorecard
```

**v1 friction points — resolved:**
1. ~~Trust info buried at bottom of event detail~~ — **FIXED.** Trust is now its own tab. One click away. Clean separation.
2. ~~"Pending" everywhere in Trust section~~ — Still shows "Pending" for T+5/T+20 moves, but now with context: "Outcome tracking has not produced a usable T+5 or T+20 move yet." This is better than raw "Pending."
3. ~~Scorecard tab duplicated UI~~ — **FIXED.** No duplicate rendering.

## Cognitive Load — 7.5/10 (was 6/10, +1.5)

### Event Detail Page: IMPROVED

v1 had 11 sections dumped on a single scroll. v2 distributes content across three tabs:

**Summary tab:**
- Header (severity, type, age)
- What Happened (catalyst summary)
- Bull Case vs Bear Case
- Similar Past Events
- Market Regime
- Quick Actions

**Evidence tab:**
- Market Context (Why It Matters)
- Source Details
- Risk Factors

**Trust tab:**
- Source Journey (pipeline visualization)
- Verification (outcome tracking)
- Feedback buttons
- Disclaimer

This is a massive cognitive load reduction. A trader scanning breaking news can read Summary in 10 seconds. If they want proof, Evidence is one tap. If they want to verify trustworthiness, Trust is one more tap. The progressive disclosure is now working as intended.

**Remaining issue:** Market Regime appears on the Summary tab AND was previously on the Evidence tab. In my testing it appeared only on Summary, which is correct. But "Quick Actions" (View original, Share) appears at the bottom of Summary — these should arguably be in the header, not at the bottom after scrolling past 5 sections.

### Feed Page: IMPROVED

Smart Feed adds intelligence to the firehose. With 3 events in Smart Feed vs potentially hundreds in All Events, the cognitive load is dramatically lower for the primary use case. The daily briefing card at top provides immediate context: "1 event detected in the last 24h for your watchlist" — this is exactly the at-a-glance status that was missing in v1.

### Scorecard Page: IMPROVED

Bucket tables are now collapsible (default collapsed except Signal buckets). Advanced Analytics is behind a collapsible section. This is correct progressive disclosure — the overwhelming wall of tables from v1 is now manageable.

## Interaction Patterns — 8/10 (was 7/10, +1)

**What improved:**
- **Feed mode switcher** changed from a dropdown to a button that expands into 3 inline options (Smart Feed / My Watchlist / All Events). This is better than a dropdown — options are visible on click and self-describing.
- **Event detail tabs** (Summary/Evidence/Trust) are clean tab buttons with clear labels. Switching is instant with no URL change (stays on same page).
- **Daily Briefing dismiss** — single click, persistent. Clean interaction.
- **Keyboard shortcuts** remain solid: j/k, Enter, /, Escape all work as expected.

**What still doesn't work:**
- **No pull-to-refresh on mobile.** For a real-time feed, this remains a missing gesture.
- **Event search results are unclickable** (404 bug). The search interaction itself is excellent — typing "oil" returns relevant events with severity badges. But the last step (click to navigate) is broken.
- **Filter interaction in feed** — the "+ Add filter" button's options are still not obvious. What can I filter BY? The History page has clear filter dropdowns (severity, source, date range, ticker). The Feed should match.

## Empty States — 6/10 (was 5/10, +1)

**Improved:**
- "Scanning 15 sources for your watchlist…" — shown when no events match current mode/filters. Better than a blank page.
- "Quiet day" concept exists in Smart Feed when filtered events produce no results.
- "No similar past events found for this ticker" — honest and clear.
- "Outcome tracking has not produced a usable T+5 or T+20 move yet" — much better than raw "Pending."

**Still missing:**
- No empty state for "0 watchlist tickers" scenario.
- No "reached the end" indicator in feed infinite scroll.

## Error Handling UX — 5.5/10 (was 4/10, +1.5)

**Push notification denial:** Major improvement. The Settings page now shows:
1. "Browser notifications are blocked for this site"
2. Step-by-step visual guide: "Click the lock icon → Find Notifications → Change to Allow → Refresh"
3. "Or get alerts via email instead (coming soon)"

This is excellent error recovery UX. The only issue is the raw "permission denied" text still appearing above the guidance.

**WebSocket status:** Now shows Connected/Reconnecting/Offline with `role="status"` and aria labels. The "Live" indicator changes state visually. This was completely missing in v1.

**Search 404:** When clicking an event search result, users hit a bare React Router error page: "404 Not Found" with a developer-facing message. This needs a custom error boundary.

## Top 5 Remaining UX Issues (Ranked by User Impact)

### 1. Fix event search result navigation (404)
The search UX is excellent. Typing "oil" returns 10+ relevant events with severity badges. But clicking any result crashes to a 404. This is the most jarring UX break in the product — it violates the user's expectation at the moment of highest engagement.

### 2. Add per-event outcome follow-up
Users see "BEARISH" on AAL → check back 5 days later → no information about what happened to AAL. The Trust tab shows "Pending" for T+5/T+20. Close the loop. Show the price movement post-alert. This is the #1 UX gap for building trust and retention.

### 3. Add fallback notification channels in Settings
The push notification recovery guide is excellent. But if the user can't/won't fix browser permissions, there's no alternative. Add email digest, Discord webhook, or Telegram options directly in Settings. The "coming soon" label on email is promising but not shipped.

### 4. Remove raw "permission denied" text from Settings
The recovery guidance makes the raw text redundant. Remove "permission denied" / "Permission: denied" and let the human-friendly "Browser notifications are blocked" message and recovery steps speak for themselves.

### 5. Improve feed filter discoverability
The "+ Add filter" button on the Feed doesn't clearly communicate what filter options exist. The History page has excellent filter UX (dropdowns for severity, source, date range, ticker). Bring the same pattern to the Feed, or at least show a preview of available filters when "+ Add filter" is clicked.
